Frames

In HTML (1)

  • Create frame1.html: 
    <!DOCTYPE html PUBLIC "-//W3C//DTD XHTML 1.1//EN"
"http://www.w3.org/TR/xhtml11/DTD/xhtml11.dtd">
<html xmlns="http://www.w3.org/1999/xhtml" xml:lang="nl">
<head>
  <title> Testpagina </title>
  <meta http-equiv="Content-Type" content="text/html;
    charset=ISO-8859-1">
</head>
<body>
  <p>...your content for frame 1 here...</p>
</body>
</html>

In HTML (3)

  • Put together in index1.html: 
    <!DOCTYPE HTML PUBLIC "-//W3C//DTD HTML 4.01 Frameset//EN"
"http://www.w3.org/TR/html4/frameset.dtd">
<html>
<head>
  <title> Testpagina </title>
</head>
<frameset rows="100,*">
  <frame src="frame1.html">
  <frame src="frame2.html">
  <noframes>
    <body>
      Sorry, this page is not for you.
    </body>
  </noframes>
</frameset>
</html>

In HTML (5)

  • Put together in index2.html: 
    <!DOCTYPE HTML PUBLIC "-//W3C//DTD HTML 4.01 Frameset//EN"
"http://www.w3.org/TR/html4/frameset.dtd">
<html>
<head>
  <title> Testpagina </title>
</head>
<frameset rows="100,*">
  <frame src="frame1.html" scrolling="no">
  <frameset cols="200,*">
    <frame src="frame2.html" scrolling="no">
    <frame src="frame3.html">
  </frameset>
  <noframes>
    <body>
      Sorry, deze pagina is niet voor jou bedoeld.
    </body>
  </noframes>
</frameset>
</html>

Frames suck

  • Not valid in HTML5
  • Frames suck!
    • index has no content → bad for SEO
    • unnecessary markup
    • multiple files mess
    • perfect CSS alternatives exist
    • unable to bookmark individual pages
    • backbutton broken
    • hard to 'read' for vision impaired
    • unsolvable scrollbar problem (demo)
    • ...
  • frames are strictly forbidden!

IFrames

In HTML

  • Embed another page with <iframe>
  • Whether the embedded page is show or not depends on the server security settings of the embedded page. Try replacing the src with e.g. http://www.w3schools.com, and inspect the console: 
    <!DOCTYPE html>
<html lang="nl">
<head>
  <title> Testpage </title>
  <meta charset="utf-8">
</head>
<body>
  <h3><a id="chapter1">Chapter 1</a></h3>
  <p>
    Lorem ipsum dolor sit amet, consectetur adipisicing elit,
    sed do eiusmod tempor incididunt ut labore et dolore magna
    aliqua. Ut enim ad minim veniam, quis nostrud ...
  </p>
  <iframe src="http://www.standaard.be" width="500" height="350"></iframe>
  <p>
    Lorem ipsum dolor sit amet, consectetur adipisicing elit,
    sed do eiusmod tempor incididunt ut labore et dolore magna
    aliqua. Ut enim ad minim veniam, quis nostrud ...
  </p>
</body>
</html>

IFrames usually suck

  • Still valid in HTML5
  • Mainly to solve technical issues:
    • the buttons in this course material use iframes under the hood
    • also used to embed Youtube videos
  • Security issues: some notorious hacks make use of an iframe
  • do not use iframes just for creating a panel with scrollbars! Use a <div> with CSS overflow: scroll instead
