  • <!DOCTYPE html>
<html lang="en">
<head>
  <title> Testpage </title>
  <meta charset="utf-8">
</head>

Basic page structure (3)

  • Every HTML document requires at least following parts:
    • Doctype declaration: this sets the document type and version
      → for HTML5 it is <!DOCTYPE html>
    • <html lang="..."> element: this is the actual HTML document; the lang attribute should contain a correct value (en for English, nl for Dutch...) to aid screen readers
    • <head> element: contains all meta information (most of it is optional): title, characterset, meta information, author, description, keywords, linked CSS or Javascript files, favicon...
    • <title> element inside the head: it should contain a relevant page title; it shows up on browser tabs and is important for search engines
    • <body> element: contains the page content with all the titles, texts, images, videos...
  • always check if your page validates on https://validator.w3.org/!

HTML elements

  • HTML elements consist of tags and add structure to content:
    • most are non-empty elements: <xxx>...</xxx>; they mark content
    • some are empty elements: <xxx>, e.g. <br>, <img>...; they insert content
  • The HTML5 standard:
    • is case insensitive, e.g. <img>, <IMG> or <Img> are the same
    • doesn't require some non-empty elements to be closed, e.g. <p>... is allowed instead of <p>...</p>
    • for historical reasons, a trailing slash is allowed in empty elements, e.g <img> is the same as <img>
  • House rules:
    • use lowercase for element names
    • always close non-empty elements
    • whether to use the trailing slash in empty elements is up to you
  • For an excellent overview see w3schools.com

Blocklevel and inline

  • Blocklevel: is rendered as a block, i.e. the other content will appear above/below
  • Inline: is rendered as text content, i.e. in horizontal lines
  • Blocklevel or inline?
    <h1>, <h2>, <p>, <a>, <ul>, <li>, <em>, <img>
  • Nesting rules:
    • Blocklevel elements can contain blocklevel and inline elements
    • Inline elements can only contain inline elements
    • Additional nesting restrictions: <p> cannot contain <ul>, <ul> can only contain <li> etc...

Attributes

  • Additional properties of an element
  • Should be in the opening tag, never in the closing tag
  • <img src="..." alt="...">
<a href="...">...</a>
  • The HTML5 standard:
    • attributes are case insensitive
    • you may use single, double or even no quotes around attribute values
  • House rules:
    • use lowercase for attribute names
    • use double qoutes around attribute value

Metatags

  • Provides metadata about the HTML document.
  • It usually consists of name/value pairs: 
    <head>
  ...
  <meta name="description" content="Slides for Webtechnology1">
  <meta name="keywords" content="HTML, CSS">
  <meta name="author" content="Rogier van der Linde">
</head>
  • Many such tags exist, in fact, you may specify any name/value pair
  • Description and keywords were once important for search engine indexing, but are largely ignored these days

[Link relations]

  • Links the document to other resources: 
    <head>
  ...
  <link rel="icon" href="img/icons/favicon.ico">
  <link rel="stylesheet" href="css/main.css">
</head>
