Elements <div> and <span>

  • Traditionally, these elements were used to mark parts of a document:
    <div>
    <span>
  • <div> is a blocklevel element (like paragraphs)
  • <span> is an inline element (like characters)

SECTIONS

Sectioning — new style

Sections and headings

  • a section is a part of the document with a heading
  • a heading (<h1>, <h2>, <h3>...) is the title of a section (not to be confused with header <header>, see later)
  • don't use a heading just because a text is large; from the w3c website:
    h1–h6 elements must not be used to markup subheadings, subtitles, alternative titles and taglines
  • a section must contain exactly one heading, and vice versa

Sectioning elements

  • HTML5 comes with four new sectioning elements:
    <section>
    <nav>
    <article>
    <aside>
  • The <body> element is also a sectioning element
  • these elements should contain one heading each
  • there's still a some debate on how to use them, but the latest W3C HTML5.3 editor draft looks solid and usable enough

Non-Sectioning elements

  • Additional elements you can use to group content without creating a section:
    <div>
    <header>
    <footer>
    <main>
  • these elements do not correspond with a heading

<section>

  • marks a thematic group of content that contains a heading
  • use for:
    • chapters of a tutorial
    • tabs of a tabbed content
    • introduction, news items and contact information areas on the homepage
    • comments area of an article
    • basically anything that contains (usually starts with) a heading
  • don't use:
    • as hook for styling or scripting (use <div> instead)
    • if <aside>, <nav> or <article> is more appropriate

Quotes from the W3C:

A general rule is that the section element is appropriate (...) if the element's contents would be listed explicitly in the document's outline.
The section element is not a generic container element. When an element is needed only for styling purposes or as a convenience for scripting, authors are encouraged to use the div element instead.

<article>

  • is a special case of <section>
  • independent content reusable in another context or website
  • use for:
    • a blogpost, tutorial or newspaper article
    • a tweet
    • a comment
    • an advertisement or banner
    • a widget: poll, calendar, latest comments...
  • don't use for:
    • a part of a blogpost
    • a login form
    • content that loses its meaning without the surrounding content
  • if not sure, use <section>

Literally from the W3C:

The article element represents a complete, or self-contained, composition in a document, page, application, or site and that is, in principle, independently distributable or reusable, e.g. in syndication. This could be a forum post, a magazine or newspaper article, a blog entry, a user-submitted comment, an interactive widget or gadget, or any other independent item of content.

<nav>

  • is a special case of <section>
  • marks the main navigation blocks in a page
  • contains a group of links only: no single links, forms, filters...
  • use for:
    • main and submenu
    • breadcrumb
    • paging
    • group of contextual links
  • don't use for:
    • language menu
    • filters, facet navigation
    • search form
    • hash tags, tag clouds
    • footer links
    • calls to action

Literally from the W3C:

The nav element represents a section of a page that links to other pages or to parts within the page: a section with navigation links.
Not all groups of links on a page need to be in a nav element — the element is primarily intended for sections that consist of major navigation blocks.

<aside>

  • is a special case of <section>
  • piece of non-essential content related to the parenting section
  • use for:
    • related advertising, links or products
    • 'recenty viewed' or 'others also recommend' sections
    • short background information or quick facts
    • a blogroll
    • a pull quote
  • don't use for:
    • non-related advertising
    • major navigation blocks

Literally from the W3C:

The aside element represents a section of a page that consists of content that is tangentially related to the content around the aside element, and which could be considered separate from that content. Such sections are often represented as sidebars in printed typography.
The element can be used for typographical effects like pull quotes or sidebars, for advertising, for groups of nav elements, and for other content that is considered separate from the main content of the page.

<div>

  • <div> is still used, but for layout, styling or scripting only
  • use for:
    • positioning and styling fragments: page or main content wrapper, cells in a grid layout...
    • container for scripts like google map
  • don't use:
    • where another more semantic element would be more appropriate: <section>, <p>, <blockquote>, <figure>, <header>, <nav>...

<header>

  • marks introductory content at the top of a section
  • may contain:
    • a heading
    • navigation elements
    • a short introduction or summary
    • a logo
    • information about the writer, post date...

Literally from the W3C:

The header element represents introductory content for its nearest ancestor sectioning content or sectioning root element. A header typically contains a group of introductory or navigational aids.
When the nearest ancestor sectioning content or sectioning root element is the body element, then it applies to the whole page.

<footer>

  • marks closing content usually at the bottom of a section
  • may contain:
    • paging, further reading and other navigation elements
    • short conclusion
    • information about the writer, copyright

Literally from the W3C:

The footer element represents a footer for its nearest ancestor sectioning content or sectioning root element. A footer typically contains information about its section such as who wrote it, links to related documents, copyright data, and the like.

<main>

  • specifies the main content of the document
  • may be used by blind browsers to skip navigation etc...
  • beware:
    • not more than one <main> element per document
    • should not contain content repeated across pages, like menus, sidebars, search forms, copyright info...
    • cannot be nested in <article>, <aside>, <nav>, <header> or <footer>

Literally from the W3C:

The main content of the body of a document or application. The main content area consists of content that is directly related to or expands upon the central topic of a document or central functionality of an application.

SECTIONS

Sectioning — Howto

Headings and sections

  • remember:
    • <body>, <section>, <article>, <nav>, <aside> are sections
    • <div>, <header>, <footer>, <main> are not sections
    • a section should have a heading
    • a heading should correspond to a section
  • at the end we should get a nice outline (plugin or online tool):

Not all elements contribute to the outline: <blockquote>, <details>, <dialog>, <fieldset>, <figure> and <td> are considered sectioning roots that have their own outlines.

Headings and sections

  • Here's how to do proper sectioning:
    • step 1: create an outline for your document (on paper!)
      • the site title is the root of the outline
      • if a heading is missing on the screenshot but you need it in the outline, choose a good one yourself (this is called a structural heading)
    • step 2: add <section> elements and headings to your HTML document
      • each outline entry corresponds to a <section> with a heading <hn>
      • the <body> is the root section with a <h1> (the logo) as heading
      • the nesting of the <section> elements in HTML is the same as in the outline
      • the heading number in HTML is the same as the outline indent level
    • step 3: where appropriate, change section to article, nav or aside
  • Additional optional steps:
    • step 4: in external sections such as ads or FB comments, restart heading numbering from <h1>
    • step 5: add <header> and/or <footer> to one or more sections
    • step 6: mark the main content with <main>
    • step 7: add <div>'s as CSS or Javascript hooks
    • step 8: apply implicit sections (see further)

Structural headings

  • Headings that appear in the outline but not on the final design are called structural
  • They should be hidden with CSS: 
    ...
<style>
  .structural {
    position: absolute;
    left: -9999px;
  }
</style>
</head>
<body>
...
<section>
  <h2 class="structural">Submenu</h2>
  ...
</section>
...
  • tip: meaningful names like 'related', 'legal info' etc... are preferred over structural names like 'sidebar', 'footer' etc...

Notice how the use of section means that the author can use h1 elements throughout, without having to worry about whether a particular section is at the top level, the second level, the third level, and so on.

Step 8 — Implicit sections (1)

  • Here's a little secret: you don't have to specify a <section> if you don't want to; a new heading autmatically starts a new section. This is called an implicit section. With this we can simplify a few things.
  • If a <section> is the only child of its parent for example: 
    ...
<footer>
  <section>
    <h2 class="structural">Footer</h2>
    ...
  </section>
</footer>
...

    ...
<footer>
  <h2 class="structural">Footer</h2>
  ...
</footer>
...

  • Or if sections are just small parts of a single content...

Step 8 — Implicit sections (3)

  • ... you may simplify it to: 
    ...
<h4>Veggies es bonus</h4>
...
<h4>Turnip greens</h4>
...
<h4>Nori grape</h4>
...
  • general rule of thumb: if you don't need it as a future style hook, you may consider removing the <section>
  • check if your outline is still correct!

It must be said that the W3C does not encourage implicit sections. A quote:

Authors are also encouraged to explicitly wrap sections in elements of sectioning content, instead of relying on the implicit sections generated by having multiple headings in one element of sectioning content.

Grouping content

Grouping content (1)

things that belong together, should be grouped together

Grouping content (2)

  • Examples of things that belong together:
    • controls, title, help texts etc.. of a single form
    • group of social media icons
    • avatar, title, date, text etc... of a single comment
    • picture and caption
    • contact information
    • introductory content: logo, top menu, login...
  • They should be wrapped in a common parent:
    • <form>
    • <ul>
    • <article>
    • <figure>
    • <dl>, <p> or <address>
    • <header>

Example: bottom menu

  • wrong: 
    <a href="#">about</a> | <a href="#">disclaimer</a> | <a href="#">contact</a>
  • better: 
    #bottommenu li + li:before {
  content: "|";
  margin: 0.5em;
}

    <ul id="bottommenu">
  <li><a href="#">about</a></li>
  <li><a href="#">disclaimer</a></li>
  <li><a href="#">contact</a></li>
<ul>
