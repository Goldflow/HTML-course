Your toolbox

What you need

  • For HTML/CSS, you don't need much:
    • a decent text editor
    • a browser (Chrome) with good development tools
    • other popular browsers (Edge, Safari, Firefox) for testing
    • [optional] some extra browser extensions
    • [optional] a web server
    • [optional] a versioning tool, e.g. github.com

Text editor

  • Any text editor will do, but it will be your biotope for many hours, so pick a decent one. Look for these features:
    • nice-looking and slick GUI
    • files & folders: navigation bar, projects, tabbed files
    • editing: code folding, syntax highlighting, autocompletion and -indentation, on-the-fly syntax check, shortcuts, refactoring, auto layout...
    • search/replace: basic search, searching in folders, regular expressions
    • customizable: color schemes, plugins, preferences
    • integrations: Git
  • most popular in web dev: Visual Studio Code

Browser

  • Develop on the best browser (which is Chrome). When everything works, test/fix for the others.
  • Start using Chrome's built-in development tools

Developer tools

  • Open with Shift-Ctrl-I or F12. Features:
    • Elements: inspect HTML & CSS
    • Console: warnings and errors are shown here
    • Sources: find all resources (fonts, images,...) in this page
    • Network: list of all network activity during page load
    • Performance: record load times, CPU & memory load...
    • Memory: view memory stack
    • Application: manage cookies, sessions, storage...
    • Security: security info about connection, certificate...
    • Audit: audit & benchmark your page, get technical tips

Browser extensions

  • Some Chrome extensions you might consider:
    • probably the best addition to the built-in devtools: web developer toolbar
    • integrated validation with HTML validator
    • inspect sectioning with HTML5 outliner
    • color picker by colorzilla
    • if you quickly need a simple web server: 200 ok

Web server

  • You don't need a web server in this course. However, some extensions or extension functions won't work without. You can install a full-blown developer stack like XAMPP or WAMP (you will need it in later courses anyway), or just use the 200 ok extension for chrome for now.

Github versioning

  • Versioning: keep track of file changes in a historical timeline
  • Github is by far the most popular versioning system. Terminology:
    • versioned files belonging to a project: repository
    • create a local copy of a remote project: clone
    • create a new project based on an existing or someone else's project: fork
    • create variations on a project for experimentation: branch
    • suggest bug fixes and improvements: pull request
    • merge two different versions: merge
    • collaborate and add your contributions as a team member: commit
  • Github can be used alone or in team, online and/or offline.

Course info

Course goals

  1. Learn to create qualitative static HTML/CSS websites with code that
    • actually validates without errors
    • is fast and optimized for search engines
    • is clean and has a proper layout
    • uses the right HTML elements in the right places
    • is mobile-friendly and responsive
  2. Acquire a basic understanding of user experience
    • accessibility
    • usability
  3. Develop business competences
    • communicating with customers
    • understanding and translating their needs
    • documentation
    • planning

Course structure

  • Your tasks
    • 12 classes of 4h with mix of theory and excercises
    • complete certain assignments at home weekly
    • study theory each week by the next class
    • create a website, preferably for a real customer, and publish online
  • My help
    • build and maintain this online course
    • give weekly assignments & discuss model solutions
    • provide tips & tricks, feedback and guidance during classes
    • email: for personal questions only (please allow 3 working days response time)
  • Evaluation
    • 50% website project
    • 50% exam

House rules

  • In order to learn, you need a serene and productive environment.
  • You can't
    • arrive late or leave early without permission
    • use your laptop or PC for any other purpose than this course
    • use any social media, including Facebook, Youtube, Snapchat, Tinder...
    • use your smartphone too frequently (and please: put it in silent mode)
    • walk around the classroom (if you need to relax, take a small break outside)
    • eat in the classroom (if you have a sugar dip, eat a snack outside)
    • copy or send code to a fellow student, not even a single line!
  • You can
    • discuss solving strategies with your fellow students
    • listen to music on your headphones
    • raise your hand to ask for help any time
    • drink, as long as it is in a closable container (no cans!)
    • use your own laptop
