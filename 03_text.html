Blocklevel elements

Structuring text

  • Avoid plain text without structure in the <body>: 
    ...
<body>
  Hello!
</body>
</html>
  • Better: 
    ...
<body>
  <p>Hello!</p>
</body>
</html>

Headings

  • Are titles and subtitles of a page: 
    ...
<h1>Title level 1</h1>
<h2>Title level 2</h2>
<h3>Title level 3</h3>
<h4>Title level 4</h4>
<h5>Title level 5</h5>
<h6>Title level 6</h6>
...
  • <h1> is normallly used for the site title, <h2> for the page title, <h3> for page chapters, <h4> is used for page subchapters etc...
  • don't use a heading just because a text is large; from the w3c website:
    h1â€“h6 elements must not be used to markup subheadings, subtitles, alternative titles and taglines

Strong and emphasize (1)

  • Compare strong with louder voice (important term), and emphasize with slower voice (new term): 
    ...
<p>
  The Congolese word <em>Ilunga</em> means that someone
  can forgive a <strong>first</strong> incident,
  accept a <strong>second</strong>, but cannot
  tolerate a <strong>third</strong> incident
</p>
...

Strong and emphasize (2)

  • do not use <strong> or <em> on a whole sentence,
    do not use <strong> just because it should look bold,
    do not use <em> just because it should look italic: 
    ...
<p><strong>Lorem Ipsum</strong></p>
<p>
  Lorem ipsum dolor sit amet, consectetur adipisicing elit, sed do
  eiusmod tempor incididunt ut labore et dolore magna aliqua. Ut
  enim ad minim veniam, quis nostrud exercitation ullamco laboris
  nisi ut aliquip ex ea commodo consequat.
  ...
</p>
<p><em>published March 2001</em></p>

Small, b and i

  • Small: 
    ...
<p>
  <small>copyright 1999-2050 by Rogier van der linde</small> incident
</p>
...
  • The tags <b> and <i> used to mean 'bold' and 'italic'. This is presentational and not allowed in HTML, so they got a different meaning in HTML5:
    • <b> now means 'stylistically offset' (if extra importance: use <em>)
    • <i> now means 'alternate voice or mood' (if extra importance: use <strong>)
  • since this causes confusion, <b> and <i> is not allowed in this course

HTML character entities

Code confusion

  • Prevent possible confusion with markup code: 
    ...
<p>
  if p1<<p2 or p1>>p2 then...
</p>
...
  • these should be encoded, see list at w3schools.com; corrected: 
    ...
<p>
  if p1&lt;&lt;p2 or p1&gt;&gt;p2 then...
</p>
...

[Character substitution]

  • Prevent character substitution when viewing page on other platforms: 
    ...
<ul>
  <li>Andreas Klöden</li>
  <li>José Azevedo</li>
  <li>Çédric Chavanel</li>
</ul>
...
  • corrected: → no longer necessary if you use UTF-8 
    ...
<ul>
  <li>Andreas Kl&ouml;den</li>
  <li>Jos&eacute; Azevedo</li>
  <li>&Ccedil;&eacute;dric Chavanel</li>
</ul>
...

Special characters

  • Special characters: 
    ...
<p>
  buy the Samsung&reg; Galaxy today for &euro;499!
</p>
...
  • → helpful for characters that are hard to find on your keyboard

Correct typography

  • Improve up your documents with the correct typographic characters: 
  • <dl>
  <dt>en dash</dt>
  <dd>
    wrong: Antwerp-Brussels<br>
    good: Antwerp&ndash;Brussels<br>
  </dd>
  <dt>em dash</dt>
  <dd>
    wrong: I was in - what is the name of that caf&eacute; again?<br>
    good: I was in &mdash; what's the name of that caf&eacute; again?
  </dd>
  <dt>minus</dt>
  <dd>
    wrong: 1+4-2=3<br>
    good: 1+4&minus;2=3<br>
  </dd>
  <dt>single quotes</dt>
  <dd>
    wrong: 'test'<br>
    good: &lsquo;test&rsquo;<br>
  </dd>
  <dt>double quotes</dt>
  <dd>
    wrong: "test"<br>
    good: &ldquo;test&rdquo;<br>
  </dd>
  <dt>ellipsis</dt>
  <dd>
    wrong: test...<br>
    good: test&hellip;<br>
  </dd>
</dl>
