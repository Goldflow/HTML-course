When to use

Tables are just tables

  • Used for data that loose their meaning without the rows and colums of tables
  • Do not use for positioning content, that's what we will use CSS for!

Table or not? (1)

  • yes; content loses meaning without a table

Table or not? (2)

  • no; there are other non-tabular desings possible

Table or not? (3)

  • no; there are other non-tabular desings possible

Table or not? (4)

  • yes; a shopping cart is a kind of spreadsheet, which is a table

[Thead, tfoot and tbody]

  • Version with <thead>, <tfoot> and <tbody>: 
    <table>
  <caption>...table caption...</caption>
  <thead>
    <tr>
      ...table header content
    </tr>
  </thead>
  <tfoot>
    <tr>
      ...table footer content
    </tr>
  </tfoot>
  <tbody>
    <tr>
      ...table data row
    </tr>
    <tr>
      ...table data row
    </tr>
  </tbody>
  <tbody>
      ...
  </tbody>
</table>
  • don't use unless you have good reasons (e.g. complex design)
